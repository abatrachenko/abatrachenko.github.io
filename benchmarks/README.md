# Benchmarks

Reference library of best-in-class sites we pattern-match against. Design and copy decisions cite benchmarks — we don't invent from scratch.

## Structure

One folder per site:

```
benchmarks/
  <site-slug>/
    analysis.md       # what we learn from it
    screenshots/      # captured pages (hero, proof, pricing, FAQ, CTA)
    url.txt           # the live URL and the date we captured
```

## Analysis template

Each `analysis.md` follows this format:

```markdown
# <site-name>

**URL:** https://example.com
**Captured:** YYYY-MM-DD
**Category:** SaaS / Agency / Consultancy / Solo-expert / Adjacent

## Why this site

One paragraph. What problem does it solve well? Why is it relevant to resonanceseo.com?

## Hero
- Headline approach: [direct-benefit / provocative-question / category-name / outcome-promise / name+what]
- Subhead strategy: [expand-promise / differentiate / handle-objection / proof-stat]
- CTA: [primary label, placement, visual treatment]
- Supporting visual: [product shot / portrait / abstract / motion / none]

## Proof
- Social proof format: [logos / testimonials / numbers / press / case-links / all]
- Placement: [above fold / mid-page / footer]
- What makes it credible

## Offer structure
- How they package what they sell
- Pricing visibility [yes / tiered / "contact" / none]

## FAQ / objections
- Which objections do they handle? Which do they dodge?

## Design system observations
- Type: [sans / serif / slab / mix]
- Color: [palette, contrast strategy]
- Motion: [on load / on scroll / hover-only / none]
- Density: [tight / medium / airy]

## What we'd steal
- Pattern A — why it works
- Pattern B — why it works

## What we'd leave
- Thing C — why it wouldn't fit our ICP/brand
```
