# Conversion

Strategic layer. Positioning, ICP, offer, proof, CTAs, A/B hypotheses. Design and copy derive from these documents — if this layer is wrong, beautiful design won't save it.

## Files

- **`positioning.md`** — ICP, promise, differentiator, voice, offer. The brief every design and copy decision checks against. START HERE.
- **`proof-inventory.md`** — every credible piece of proof the user owns (clients, case studies, numbers, testimonials, credentials). Nothing goes on the site that isn't inventoried here.
- **`offer-canvas.md`** — what the user actually sells: packages, deliverables, timeline, price range (even if not public), objections handled.
- **`ab-hypotheses.md`** — ranked list of conversion experiments to run post-launch. Each entry: hypothesis, metric, expected lift, cost to run.
- **`cta-ladder.md`** — the sequence of commitment asks: newsletter → resource download → discovery call. Maps CTAs per section.

## Workflow

1. **Positioning first.** No design decisions without `positioning.md` filled in.
2. **Proof inventory second.** The hero and social-proof sections are constrained by what can actually be claimed.
3. **Offer canvas third.** Shapes the mid-page sections (what/how/for-whom).
4. **Then design.** Track B phase 3 onward.
5. **Then test.** A/B hypotheses feed Track B phase 5 (post-launch optimization).

## Update cadence

- `positioning.md` — revisit quarterly or after major client wins that reshape positioning
- `proof-inventory.md` — update whenever a new credible data point becomes available (new client, new case study, new press mention)
- `offer-canvas.md` — update when pricing or deliverables change
- `ab-hypotheses.md` — living queue, reorder as tests run and data comes in

## Voice / copy guardrails

Every line of copy on resonanceseo.com must pass:

- **Specific.** Concrete outcomes, not vague benefits.
- **Provable.** Claims backed by `proof-inventory.md`.
- **For the ICP.** Written to one specific person, not a generic buyer.
- **One job per sentence.** If a sentence could be cut without losing meaning, cut it.
- **Human, not AI-generic.** Matches the user's voice (captured in `positioning.md` once written).

Every section on the site must pass:

- **One conversion or credibility goal.** If a section can't answer "what does this section do for the reader's decision?", it doesn't ship.
